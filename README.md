# stainless-config

Stainless SDK generation config for the [VAIF Studio API](https://api.vaif.studio).

## What's in here

- **`stainless.yml`** — main Stainless config. Maps OpenAPI operations from `https://api.vaif.studio/openapi.json` to SDK resource methods, authentication schemes, error class hierarchy, and per-target package metadata.
- **`openapi.json`** — frozen snapshot of the production OpenAPI 3.1 spec. Stainless reads this on every build trigger; updating it triggers regeneration.

## How to update

The OpenAPI spec is the single source of truth and lives in the platform monorepo at `apps/api/openapi.json`.

```bash
# 1. Regenerate the Stainless config from the live spec
cd /path/to/vaif-studio
pnpm exec tsx scripts/codemod/generate-stainless-config.ts

# 2. Copy the produced files into this repo
cp stainless.yml /path/to/stainless-config/stainless.yml
cp apps/api/openapi.json /path/to/stainless-config/openapi.json

# 3. Commit + push
cd /path/to/stainless-config
git add stainless.yml openapi.json
git commit -m "chore: refresh Stainless config from OpenAPI spec"
git push origin main
```

Stainless picks up changes on its next polling cycle (typically <5 minutes), or you can trigger a regeneration manually from the Stainless dashboard.

## SDK targets

| Target | Package | Repo | Registry |
|---|---|---|---|
| TypeScript | `@vaif/client` | [VAIF-TECH/vaif-js](https://github.com/VAIF-TECH/vaif-js) | npm |
| Python     | `vaif`         | [VAIF-TECH/vaif-python](https://github.com/VAIF-TECH/vaif-python) | PyPI |
| Swift      | `VaifClient`   | [VAIF-TECH/vaif-swift](https://github.com/VAIF-TECH/vaif-swift) | SwiftPM (via tagged GitHub releases) |
| Go         | `vaif-go`      | [VAIF-TECH/vaif-go](https://github.com/VAIF-TECH/vaif-go) | Go modules (via tagged commits) |

## Hand-written code (not generated by Stainless)

Each SDK has hand-written modules for realtime (WebSocket channels) and storage (multipart upload + resume). These live behind path allow-lists Stainless cannot touch:

- **TypeScript**: `src/realtime/**`, `src/storage/**`, `src/index.ts`
- **Python**: `src/vaif/realtime/**`, `src/vaif/storage/**`
- **Swift**: `Sources/VaifClient/Realtime/**`, `Sources/VaifClient/Storage/**`
- **Go**: `realtime/`, `storage/`

## Tooling packages (not Stainless-generated)

These four are hand-written tools that consume the SDK rather than being generated alongside it:

- [`@vaif/cli`](https://github.com/VAIF-TECH/vaif-cli) — type generation + dev tooling
- [`@vaif/react`](https://github.com/VAIF-TECH/vaif-react) — React / React Native hooks
- [`@vaif/mcp`](https://github.com/VAIF-TECH/vaif-mcp) — MCP server for Claude Code / Cursor
- [`@vaif/migrate`](https://github.com/VAIF-TECH/vaif-migrate) — codemod for migrating from `@vaiftech/*` to `@vaif/*`
